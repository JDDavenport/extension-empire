# Extension Launch Checklist

Use this for every new extension.

## Pre-Build
- [ ] Gap identified (competitor weakness or unserved niche)
- [ ] Clear free/pro split defined
- [ ] Extension name chosen (keyword-rich)
- [ ] Estimated build time: ___

## Build
- [ ] manifest.json (Manifest V3)
- [ ] Core functionality working
- [ ] Popup UI (clean, dark theme, fast)
- [ ] Context menu items (if applicable)
- [ ] Keyboard shortcuts (if applicable)
- [ ] Toast notifications for user feedback
- [ ] ExtensionPay integrated (if monetized)
- [ ] Tested on 3+ websites

## Chrome Web Store Submission
- [ ] Icons: 16x16, 48x48, 128x128 PNG
- [ ] Screenshots: 1280x800 or 640x400 (at least 3)
- [ ] Store description (keyword-optimized)
- [ ] Privacy policy hosted (GitHub Pages or similar)
- [ ] Category selected
- [ ] Single purpose description (required by Google)

## Post-Launch (Week 1)
- [ ] Verify listing is live
- [ ] Check for initial reviews
- [ ] Monitor install count
- [ ] Respond to any reviews

## Post-Launch (Month 1)
- [ ] Evaluate: >50 installs? Keep. <50? Assess and potentially kill.
- [ ] Read all reviews — feature requests become v1.1
- [ ] ASO: adjust title/description based on search terms
